# プライバシーポリシー記入チェックリスト

## ⚠️ 重要：審査前に必ず記入が必要な項目

改訂版のプライバシーポリシー（`privacy_policy_revised.html`）には、以下の情報を記入する必要があります。

---

## 1. 運営者情報（必須）

### 個人開発者の場合
```html
運営者: 山崎 翔平（個人事業主）
所在地: 〒123-4567 東京都○○区○○ 1-2-3
代表者: 山崎 翔平
```

### 法人の場合
```html
運営者: 株式会社SOZO
所在地: 〒123-4567 東京都○○区○○ 1-2-3 ○○ビル4F
代表者: 代表取締役 山崎 翔平
法人番号: 1234567890123
```

**注意事項：**
- 住所は実在する必要があります（私書箱も可）
- バーチャルオフィスの住所でも問題ありません
- 個人宅の場合、プライバシーを考慮してください

---

## 2. 連絡先情報（必須）

### メールアドレス
- [ ] **サポート用メール**: support@[your-domain].com
- [ ] **プライバシー専用メール**: privacy@[your-domain].com（推奨）

### 推奨される無料メールサービス
- Gmail for Business（無料版）
- ProtonMail（プライバシー重視）
- Outlook

**例：**
```
support@sozoapp.com
privacy@sozoapp.com
```

---

## 3. データ保護責任者（DPO）

### 必要な場合
- EU/EEAユーザーが多い場合
- 大規模なデータ処理を行う場合

### 不要な場合（一般的なアプリ）
この部分は削除して構いません：
```html
<!-- DPOセクションを削除 -->
```

---

## 4. 第三者サービスの確認

現在記載されているサービスが正しいか確認：

- [ ] **Supabase**: データベース、認証（使用している）
- [ ] **OpenAI**: AI機能（使用している）
- [ ] **その他のサービス**:
  - Firebase（使用している場合追加）
  - Google Analytics（使用している場合追加）
  - その他の分析ツール

---

## 5. データ保持期間の調整

デフォルト値を確認し、必要に応じて調整：

| データ種類 | デフォルト期間 | 調整が必要？ |
|----------|------------|----------|
| アカウント情報 | 削除後30日 | □ |
| 音声データ | 24時間以内削除 | □ |
| 技術ログ | 90日 | □ |
| クラッシュレポート | 180日 | □ |

---

## 6. 年齢制限の決定

以下から選択：

- [ ] **4+（全年齢）**: 子供向けコンテンツ規制に準拠が必要
- [ ] **13+（推奨）**: COPPA回避、一般的な選択
- [ ] **17+**: 成人向けコンテンツがある場合

**現在の設定**: 13+

---

## 7. 簡易版 vs 完全版の選択

### オプションA: 簡易版を使用（小規模アプリ向け）
- 最初に作成した `privacy_policy.html` を使用
- メールアドレスのみ変更
- 個人開発者や小規模プロジェクトに適切

### オプションB: 改訂版を使用（推奨）
- `privacy_policy_revised.html` を使用
- すべての情報を記入
- より包括的で審査通過率が高い

## 実際の記入手順

### Step 1: ローカルで編集

```bash
cd sozo-app-policies
cp privacy_policy_revised.html privacy_policy.html
nano privacy_policy.html

# 以下の文字列を検索して置換：
# [会社名/個人事業主名をここに記入] → あなたの情報
# [住所をここに記入] → あなたの住所
# [代表者名をここに記入] → あなたの名前
# [実際のサポートメールアドレス] → support@example.com
# [実際のプライバシー担当メールアドレス] → privacy@example.com
```

### Step 2: 不要なセクションの削除（任意）

```html
<!-- 以下は削除可能 -->
<!-- EU代理人（不要な場合） -->
<!-- DPO情報（不要な場合） -->
<!-- CCPA/GDPRセクション（対象外の場合） -->
```

### Step 3: GitHubにプッシュ

```bash
git add privacy_policy.html
git commit -m "Update privacy policy with actual information"
git push
```

## よくある質問

### Q: 個人情報（住所）を公開したくない

**A:** 以下の選択肢があります：
1. バーチャルオフィスを利用（月額3,000円程度）
2. 私書箱を利用
3. 「お問い合わせフォームから開示」と記載

### Q: メールアドレスは Gmail でも大丈夫？

**A:** はい、問題ありません。ただし：
- 専用のアドレスを作成することを推奨
- support@、info@、privacy@ などのプレフィックスを使用
- 定期的にチェックすること

### Q: すべての項目を記入する必要がある？

**A:** いいえ。最低限必要なのは：
- 運営者情報
- 連絡先
- 収集するデータの種類
- 利用目的
- 第三者への提供

### Q: 英語版も必要？

**A:**
- 日本国内のみ: 不要
- 海外展開予定: 推奨
- App Store Global: 英語版があると良い

## 審査での注意点

### App Store
- プライバシーポリシーURLが有効であること
- データ収集の内容が一致していること
- 子供向けの場合、追加要件あり

### Google Play
- データセーフティセクションとの整合性
- 必須記載事項がすべて含まれていること
- 日本語で記載されていること

## 最終アドバイス

1. **正直に記載する**: 虚偽の情報は審査で却下されます
2. **シンプルに始める**: 最初は簡易版でも問題ありません
3. **定期的に更新**: サービスの変更に応じて更新
4. **ユーザーファースト**: 読みやすく、理解しやすく

審査は厳しくなってきていますが、基本的な情報が正確に記載されていれば問題ありません。

不安な場合は、簡易版（`privacy_policy.html`）から始めて、徐々に充実させていくことをお勧めします。
